# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.0.x | Yes |
| < 1.0.0 | No |

## Reporting a Vulnerability

Please report security vulnerabilities by emailing security@sourcebound.dev.

Include enough detail for maintainers to reproduce and assess the issue, such as affected version, configuration, impact, and a minimal reproduction when possible.

## What Not to Include in Public Issues

Do not include sensitive material in public GitHub issues, pull requests, screenshots, logs, or discussions. This includes:

- API keys
- PHI
- Credentials
- Session tokens
- Private documents or document excerpts that are not safe to disclose

## Response Timeline

Maintainers aim to acknowledge vulnerability reports within 48 hours. After acknowledgment, we will work with the reporter to validate impact, identify mitigations, and coordinate a fix or disclosure plan.
